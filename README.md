# Project Overview

This project is an e-commerce API built in Go, with a strong focus on modularity and abstraction. Database interactions are largely handled by [sqlc](https://sqlc.dev/), which generates type-safe Go code from SQL queries, minimizing boilerplate and manual query handling.

## Application Structure

The entry point is api.go, where application configuration and setup occur.

### Configuration

- **dbConfig struct**: Contains a `dsn` field, representing the database connection string.
- **config struct**: Includes the `dsn` and an `addr` field, which specifies the server port.
- **application struct**: Embeds the above configuration and a `db` field, which holds the live database connection.

The application instance (`app *application`) is used throughout the codebase. The database connection is accessed via `app.db` and passed to various components, ensuring consistent access to the database.

### Mounting Routes

Within `api.go`, the `mount()` method is defined on the `application` struct:

```go
func (app *application) mount() http.Handler {}
```

This method sets up the HTTP routes and handlers. For example:

```go
productService := products.NewService(repo.New(app.db))
productHandler := products.NewHandler(productService)
r.Get("/products", productHandler.ListProducts)
r.Post("/create-product", productHandler.CreateProduct)
```

- **NewService**: A constructor that, through several layers of abstraction, provides methods to interact with the database. It wraps the repository logic, which is generated by sqlc.
- **NewHandler**: Creates HTTP handlers that expose the service methods via REST endpoints.

### Abstractions

- **Service Interface & Constructor**: The service layer defines interfaces and constructors, abstracting business logic from database access. This separation allows for easier testing and maintenance.
- **Repository Layer**: Encapsulates all database operations. The repository is initialized with the live database connection and is used by the service layer.

## Database

All SQL queries and migrations are managed in sqlc and migrations. The sqlc tool generates Go code for queries defined in `queries.sql`, ensuring type safety and reducing manual coding errors.

## Extending the API

To add new features:
1. Define new SQL queries in `queries.sql`.
2. Run sqlc to generate Go code.
3. Create new service methods in the relevant service package.
4. Add new handlers and routes in `api.go`.

## starting the API
locally
```
go run /cmd
```

in dev container
```
go run <absolute-path/cmd>
```

## Test this API with local db
run this command to get your machine's IP-ADDRESS
```
hostname -I 
```

forward traffic from your machine to the deployed database
```
kubectl port-forward --address 0.0.0.0 svc/postgres-db -n ecom-go-db 5432:5432
```

update your devcontainer.env
```
DBSTRING=postgres://postgres:postgres123@<YOUR_HOST_IP>:5432/go_ecom
```

## Summary

This project leverages Go’s strong typing and sqlc’s code generation to create a clean, maintainable, and scalable API. The abstractions around configuration, database access, and HTTP handling make it easy to extend and test.
